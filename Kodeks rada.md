# Kodeks rada:

### 1. Klonirati glavni repozitorij na lokalno računalo
### 2. Bitno: Na lokalnom repozitoriju izmjene se rade na dodijeljenoj feature grani. 
### 3. Prema zahtjevu/dogovoru radi se push te grane na GitHub repozitorij. 
### 4. Rad u grupi podrazumijeva da član grupe koji želi objaviti svoje izmjene (push) na udaljenu feature granu, prvo mora dohvatiti udaljenu granu (pull), pregledati ako je štogod na grani u međuvremenu mijenjano od zadnjeg dohvata, napraviti spajanje svojeg koda s dohvaćenim i rješiti konflikte ukoliko postoje. Nakon toga radi lokalni commit kojim se potvrđuju izmjene i push svega na udaljeni repozitorij.
### 5. Nikako se ne radi na development ili main granama.
### 6. Kad je funkcionalnost riješena i testirana, student radi pull request u GitHubu od feature grane prema development grani, te traži code review od vlasnika repozitorija.
